# 我是如何搭建个人博客工作台的

这个博客的目标不是做一个只有文章的传统站点，而是做一个长期可维护的个人数字工作台。

它目前包含几个核心区域：

- 首页：展示个人定位和快速入口。
- 项目页：集中放置已经完成或正在维护的网站项目。
- 知识库页：作为 Notion 知识库的入口。
- 学习页：收纳刷题和学习导航。
- 文章页：沉淀建站、阅读、学习和创作记录。

## 技术选择

前端用 GitHub Pages 托管静态页面，几乎没有维护成本。

后端代理从 Netlify Function 切换到 Deno Deploy，主要是为了减少额度压力，同时保留未来接入 API 的空间。

## 下一步

后续可以继续补：

1. 文章分类和标签筛选。
2. Markdown 自动生成文章页面。
3. Deno Deploy 提供文章搜索接口。
4. Notion 数据同步到静态 JSON。

这个站点会慢慢长成一个真正属于自己的数字花园。
